# QOMISO Game Tour

自転車ロードレースをモチーフにした、選手・サポート・戦術カードを組み合わせるターン制カードバトルゲームの初期プロトタイプです。

## コンセプト

- 選手カードの能力値は固定。レベル上げ、経験値、練習、限界突破による恒久強化は行わない
- 選手カード8枚、サポートカード4枚、戦術カード最大5枚を組み合わせ、レース中の一時補正と展開を作る
- 実在機材をモチーフにした架空フレーム・ホイール・タイヤを選び、レース条件に合わせる
- ウェアはプレイヤーが色・柄・スポンサー風ロゴをデザイン/選択する
- 8人チームを編成し、主副脚質とエース／アシスト適性に応じて当日エース、副エース、ロードキャプテン、リードアウトなどを任命する
- チーム構成と年間方針によって狙うステージが変わる。総合狙いは山岳・TT・横風局面を主導し、スプリントチームは平坦ステージを重点化する
- 現在・過去のワールドクラスチームをモチーフにした架空22チームを登場させる
- 選手獲得は緩いスカウトを採用し、選手価値はCredit年俸で表す。勝つための課金ではなくチーム作りの幅を増やす課金にする

## 完全カードバトル方針

- 選手カード: 8枚。主脚質、副脚質、適性タグ（最大3つ）、エース適性、アシスト適性、固定能力、固有パッシブ、Credit年俸を持つ
- 主脚質: 最も得意な分野。能力補正、AI判断、固有パッシブの発動に強く影響する
- 副脚質: 次に得意な分野。主脚質より弱い補正で対応コースを広げる
- 適性タグ: 第3脚質ではなく、パンチ力、ステージハンター、逃げ屋など局面の得意を表す。原則最大3つ
- 固有パッシブ「万能型」: 特別な選手だけが持ち、第3分野に相当する適性タグのペナルティを副脚質に近い水準まで軽減する
- サポートカード: 最大4枚。監督、発射台、補給、メカニックなどのチーム効果を持つ
- 戦術カード: 7種から最大5枚。位置取り、牽引、アタック、エース保護、スプリント準備、補給、高速ダウンヒル
- アクティブアビリティ: レース前に1枚選び、条件を満たした最初の勝負セクターで発動する
- 勝敗はカードのレベル差ではなく、Credit制限内の編成、脚質相性、役割設定、カードの切りどころ、疲労・補給・健康度・乳酸管理で決まる
- レベル、経験値、練習、覚醒、限界突破、重複による恒久強化は実装しない
- 重複カードは能力強化に使わず、未所持カードとの契約に使える汎用契約ポイントへ変換する
## 現在の実装

- 静的HTML/CSS/JavaScriptのみで動作
- `index.html` をブラウザで開くとプレイ可能
- 戦術カード最大5枚、サポートカード4枚、選手カード8枚、当日エース指名、5種類のレースを実装
- メンタル能力としてチームワーク・エゴ・負けん気を実装
- レース種別として平坦・丘陵・山岳・個人タイムトライアル・チームタイムトライアルを実装
- レース形式として1日のワンデーレースと、21ステージを省略しないグランツール型ステージレースを実装
- 実在レースをモチーフにした架空コースを実装
- コースコンディションとして風・グラベル・石畳を実装
- バイクコントロールを独立能力として実装し、下りの速度、悪路消耗、落車確率へ反映
- 瞬発力を5〜20秒の最終加速、パンチ力を30秒〜5分の短坂出力、巡航力を数分以上の高速維持として分離
- チーム設定として22チーム構成を実装
- レース当日は当日エースを指名し、他の選手へ副エース、アシスト、ロードキャプテン、リードアウトなどの役割を割り当てる
- 機材は性能に影響し、ウェアは見た目カスタム中心で一部条件ボーナス候補
- レースバトルはリアルタイムではなく、対CPU/対人ともに距離ベースのターン制として設計
- レース中の疲労、スプリント残量、糖質・水分補給、健康度を数値化し、アタック、牽引、悪路、風、登坂、終盤勝負へ反映
- 高強度走で増える乳酸負荷と選手固有の回復力を実装し、反復アタック、丘陵、再スプリントへ反映
- チームワークによるドラフティングとエース保護、補給地点、補給不足によるハンガーノック、疲労下のスプリント低下を実装
- 200km級レースは約10km間隔を目安にしつつ、山岳入口・中間ポイント前・石畳/グラベル入口・残り局面などの重要地点でカード選択・シミュレート・次展開提示を繰り返す
- グランツールは21ステージすべてを保持し、各ステージをフル操作・重要地点だけ操作・自動シミュレートから選べる
- グランツール各ステージには、総合・スプリント・山岳・クラシック・逃げ/ステージ狙い・TT狙いの方針別重要度を設定する
- 自動シミュレートでも省略ではなく、疲労・事故・順位・ポイント変動を処理する
- 課金設計として、シーズンパス、緩い選手スカウト、見た目課金、レース拡張パックを設定する
- スカウトは確率表示、天井、無料配布、重複時の汎用契約ポイント変換を前提にする。レベル上げ、限界突破、重複強化、期間限定人権は実装しない
- ゲーム独自組織「国際ロードサイクル連合（UCR）」と、年間ポイント／ライダー順位／チーム順位／対人レートの設定パネルを実装

## レース中コンディションの考え方

- `疲労`: 距離、高強度のアタック、登坂、横風、石畳・グラベルで蓄積する。持久力とチームの風よけで軽減する。
- `スプリント`: 加速、アタック、位置取りで消費する。隊列内での温存と十分な補給で一部回復する。
- `補給`: 糖質と水分をまとめたゲーム用指標。補給地点で回復し、低下するとハンガーノック域に入り、疲労と健康度を悪化させる。
- `健康度`: その日の体調、脱水耐性、悪路消耗をまとめた指標。疲労と補給不足が深刻になると出力を制限する。
- `バイクコントロール`: コーナリング、路面追従、下りのライン選択を表す。高いほど下り区間の出力が上がり、悪路の健康度低下と落車確率が下がる。
- `瞬発力`: 5〜20秒程度の加速と最高速。ゴールスプリントと瞬間的な位置取りに使う。
- `パンチ力`: 30秒〜5分程度の高出力。短坂、丘陵アタック、集団からの飛び出しに使う。
- `巡航力`: 数分以上にわたり高速を維持する力。平坦牽引、逃げ、個人TT、チームTTに使う。
- `乳酸負荷`: アタック、短い登り、スプリントで増える高強度代謝負荷のゲーム表現。高いほど再加速とスプリント出力が落ちる。
- `回復力`: 流し、隊列内、補給、下りで乳酸負荷を下げ、スプリント残量を戻す速さ。丘陵の反復加速や登り後のフィニッシュで特に有効。
- 乳酸は単なる疲労物質ではなくエネルギー代謝にも関わるため、実測血中乳酸値ではなく複合的な高強度負荷として抽象化する。
- 現実の研究をそのまま医療シミュレーションにせず、長時間走では糖質・水分摂取が重要であり、疲労により終盤出力が低下する関係をゲーム向けに抽象化している。

## アビリティ設計

- `パッシブ`: 選手固有の自動発動能力。編成、当日エース、山岳・横風などの条件に応じて、疲労、スプリント消費、補給効率、健康度、終盤出力を補正する。
- `アクティブ`: レース前に1つ選択し、対応する最初の勝負セクターで1回だけ発動する。発動時は疲労・スプリント・補給・健康度とセクター出力が同時に変化する。
- 実装済みアクティブは「全開アタック」「エース完全保護」「チーム補給」「最終発射」「横風分断」。コースと編成に合わない能力を選ぶと不発になる。
- 選手カードには固有パッシブ名を表示し、レース画面には発動中パッシブと選択中アクティブを表示する。

## 初期選手の実在モチーフ

ゲーム内名・外見・経歴・所属は架空とし、実在選手からは脚質、役割、勝負パターンだけを研究用モチーフとして抽出する。

初期300名は、固定の「勝利担当／アシスト系」では分けない。選手固有の `主脚質`・`副脚質` と、0〜100の `エース適性`・`アシスト適性` を持たせる。

主脚質は総合型38、スプリンター53、クライマー51、パンチャー30、クラシック型67、TT・ルーラー型61。レースごとの役割は当日エース、副エース、アシスト、ロードキャプテン、リードアウト、ステージハンター、逃げ屋から任命する。ステージハンター23名、逃げ屋22名を確保する。

8人編成は「エース候補2名＋支援役割6名」を基本形とするが、支援は選手の固定身分ではなく当日の任務とする。コース、体調、疲労、健康度、年間目標によって同じ選手の役割を変えられる。

初期300名は時代区分が「近代」「現代」の選手のみとし、「歴史」「クラシック」扱いの選手は後日登場するレジェンド別枠へ分離する。脚質としてのクラシック型は初期枠にも残す。

全300名に仮Credit年俸を設定する。8人チームの年俸上限はレース格に応じて通常42,000Cr、上位45,000Cr、最高峰48,000Crとし、主副脚質、両適性、固有能力、専門役割によって個別調整する。

世代最強級カードは例外枠として15,000Crまで設定する。Tadej Pogacarモチーフはエース適性99、アシスト適性84、15,000Crとし、Mathieu van der Poelモチーフはモニュメント8勝の最上位クラシック枠として13,000Crに設定する。

グランツール総合優勝が2回以上の選手は、勝利回数に応じてエース適性96〜98、Credit10,800〜13,000の最低保証を適用する。既存値は下げず、ポガチャル型のエース適性99・15,000Crは唯一の別格枠として維持する。
