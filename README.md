# QOMISO Game Tour

2026 ツール・ド・フランス全21ステージを舞台に、選手・サポート・戦術カードを組み合わせるターン制カードバトルゲームの初期プロトタイプです。

## コンセプト

- 選手カードの能力値は固定。レベル上げ、経験値、練習、限界突破による恒久強化は行わない
- 選手8人が提供する固有カードをバトルの主役にし、別管理のチームカードと組み合わせてレース展開を作る
- 実在機材をモチーフにした架空フレーム・ホイール・タイヤを選び、レース条件に合わせる
- ウェアはプレイヤーが色・柄・スポンサー風ロゴをデザイン/選択する
- 8人チームを編成し、主副脚質とエース／アシスト適性に応じて、30種類の役割・戦術特性からエース、サブエース、牽引、保護、攻撃、位置取り役を任命する
- チーム構成と年間方針によって狙うステージが変わる。総合狙いは山岳・TT・横風局面を主導し、スプリントチームは平坦ステージを重点化する
- 現在・過去のワールドクラスチームをモチーフにした架空22チームを登場させる
- 選手獲得は緩いスカウトを採用し、選手価値はCredit年俸で表す。勝つための課金ではなくチーム作りの幅を増やす課金にする

## 完全カードバトル方針

- 選手カード: 8枚。主脚質、副脚質、適性タグ（最大3つ）、エース適性、アシスト適性、固定能力、固有パッシブ、Credit年俸を持つ
- 主脚質: 最も得意な分野。能力補正、AI判断、固有パッシブの発動に強く影響する
- 副脚質: 次に得意な分野。主脚質より弱い補正で対応コースを広げる
- 適性タグ: 第3脚質ではなく、パンチ力、ステージハンター、逃げ屋など局面の得意を表す。原則最大3つ
- 固有パッシブ「万能型」: 特別な選手だけが持ち、第3分野に相当する適性タグのペナルティを副脚質に近い水準まで軽減する
- サポートカード: 最大4枚。監督、発射台、補給、メカニックなどのチーム効果を持つ
- 戦術カード: 7種から最大5枚。位置取り、牽引、アタック、エース保護、スプリント準備、補給、高速ダウンヒル
- アクティブアビリティ: レース前に1枚選び、条件を満たした最初の勝負セクターで発動する
- 勝敗はカードのレベル差ではなく、Credit制限内の編成、脚質相性、役割設定、カードの切りどころ、疲労・補給・健康度・乳酸管理で決まる
- レベル、経験値、練習、覚醒、限界突破、重複による恒久強化は実装しない
- 重複カードは能力強化に使わず、未所持カードとの契約に使える汎用契約ポイントへ変換する

## 採用済み次期バトル設計

- 選手1人につき基本技3枚、得意技2枚、勝負手1枚の6枚を持ち、8人編成では48枚から選手別に使用する
- 個人カードとチームカードは別デッキ・別UIで管理する
- チームカードは全18種類から分類を問わず好きな6枚を選び、チームカー選択時に6枚を表示する
- 主役割を持つエース50名には二つ名と、コスト3・レース中1回の固有勝負手を付与する
- 基本技3枚と得意技2枚は選手固有にせず、地形・役割に対応する共通カードから構成する
- アシスト適性80以上の15名は強力な共通得意技を選択でき、非主エース13名には固有の支援型勝負手1枚を保証する
- チームカー選択時はチームカード、選手選択時はその選手の個人カードを表示する
- 各ターンの作戦コストは原則5。基本技1、得意技2、勝負手3、チームカード1〜2を消費する
- 1選手につき1ターン1枚まで、合計作戦コスト内なら複数選手のカードを使用できる
- 選択したカードを行動キューへ並べ、指定順で解決する。作戦コストは体力・SPとは別資源とする
- メイン画面は簡略化した俯瞰地図とし、START、GOAL、逃げ集団、メイングループ、標高、風を表示する。独立した現在地マーカーは置かない
- 左端にチームカーと選手8人を縦に並べ、各選手の常時表示は体力・SPの2本のバーに絞る
- 選択中選手のカード3枚を画面下部に表示し、複数選手の選択カードは行動キューへ並べる
- 実況はカードより下の絶対最下段に1行表示する
- 同一チームは通常時に同一ジャージと同一系統のヘルメットを着用し、現リーダー・現王者だけを例外表示する
- 詳細は `docs/battle_card_ui_design.md` を正本とする

## 現在の実装

- 静的HTML/CSS/JavaScriptのみで動作
- `index.html` をブラウザで開くとプレイ可能
- ゲーム開始時の主導線として、初回無料10連から8人チームを結成する選手スカウトを実装
- 通常単発・10連を実装。レアリティ表示は設けず、10連ではエース適性85以上の候補1人を保証
- 選手獲得時に個人カード6枚を同時解放し、重複は能力強化せず汎用契約ポイントへ変換
- レース結果に応じてスカウトCreditを獲得し、再び選手を引くゲームループを実装
- 契約済み選手をクリックすると、脚質、全15能力、役割・適性、Credit、個人カード6枚を固定表示
- チームカード18種類を実装し、ステージ前に種類を問わず好きな6枚を編成可能
- 編成した6枚はランダム抽選せず常時使用でき、地形一致、状態変化、個人カードとのコンボをレース計算へ反映
- 戦術カード最大5枚、サポートカード4枚、選手カード8枚、エース指名を実装
- メンタル能力としてチームワーク・エゴ・負けん気を実装
- レース種別として平坦・丘陵・山岳・個人タイムトライアル・チームタイムトライアルを実装
- 2026 ツール・ド・フランスの公式ルートデータを、Stage 1からStage 21までプレイ可能なメインレースとして実装
- ステージ完了後は翌ステージへ自動で進み、疲労・スプリント残量・補給・健康度・乳酸負荷を持ち越す
- 各ステージの日付、スタート／フィニッシュ、距離、獲得標高、風、リスク、コースプロフィールをレースへ反映
- コースコンディションとして風・グラベル・石畳を実装
- 各セクターへ実勾配、風向、風速、路面を設定し、登り勾配と平坦の空気抵抗を疲労へ反映
- 25km/h以上の横風平坦区間でエシュロンを判定し、チームワーク、技術、横風要員、チームカードで分断リスクを軽減
- 下り勾配ではバイクコントロールに応じて落車を、石畳・グラベルでは装備とメカニックに応じて機材故障を判定
- バイクコントロールを独立能力として実装し、下りの速度、悪路消耗、落車確率へ反映
- 選手個人の能力値はPro Cycling Manager型の50〜85尺度を採用し、50を最低、70台を一線級、80以上を世界トップ級、85を歴史的最高峰とする。85は能力ごとに最大2名、1選手につき最大2項目とする
- 世界選手権エリート男子ロードと個人TTは、2025年大会までの金・銀・銅を実績点化して能力へ補正し、その後に300名順位上限を適用する
- パヴェ能力はパリ〜ルーベのキャリア成績を主基準とし、最高順位に複数回の上位・完走実績を加味する。未出走を推測で高評価にはしない
- スプリントを5〜20秒の最高速、加速力を速度変化の鋭さ、パンチ力を30秒〜5分の短坂出力、巡航力を数分以上の高速維持として分離
- 持久力を長距離のエネルギー容量、耐性をアタックやスプリントなど高強度運動への強さとして分離
- チーム実効値はエース55%、各能力の上位アシスト2名35%、チーム平均10%を基本とする。アシスト適性と役割適合も支援効果へ反映し、8人分の能力単純合算は行わない
- チーム設定として22チーム構成を実装
- レース当日はエースを指名し、他の選手へ `data/rider_role_definitions.json` の役割・戦術特性を割り当てる
- 機材は性能に影響し、ウェアは見た目カスタム中心で一部条件ボーナス候補
- レースバトルはリアルタイムではなく、対CPU/対人ともに距離ベースのターン制として設計
- レース中の疲労、スプリント残量、糖質・水分補給、健康度を数値化し、アタック、牽引、悪路、風、登坂、終盤勝負へ反映
- 高強度走で増える乳酸負荷と選手固有の回復力を実装し、反復アタック、丘陵、再スプリントへ反映
- 連続ステージ用の日間回復力を50〜85で持たせ、前日の疲労・SP・補給・健康度・乳酸負荷から翌日開始値を生成する
- 連戦時は選手ごとの日間回復力で体力・SPを個別回復し、次ステージへ差を持ち越す
- チームワークによるドラフティングとエース保護、補給地点、補給不足によるハンガーノック、疲労下のスプリント低下を実装
- 200km級レースは約10km間隔を目安にしつつ、山岳入口・中間ポイント前・石畳/グラベル入口・残り局面などの重要地点でカード選択・シミュレート・次展開提示を繰り返す
- グランツールは21ステージすべてを保持し、各ステージをフル操作・重要地点だけ操作・自動シミュレートから選べる
- 2026 Tour de France公式発表の21ステージ（発着地・距離・種別・休息日）を参照したコースデータと、20チーム×8人・40,000 Creditの一括シミュレーターを実装
- 160選手の個人総合、全3,360件のステージ順位、勝利・表彰台・疲労・健康度・本人の落車／故障・チーム単位のエシュロン履歴を出力する
- 総合、ポイント賞、山岳賞、新人賞を個別集計し、CSV・JSON・統合レポートへ出力する。新人賞は2001年1月1日以降生まれを対象とする
- ポイント賞は公式2026コースの全19中間スプリントと各ステージの公式フィニッシュ配点、山岳賞はコース上の全69山岳頂上（4級〜超級）で通過順位を別判定して集計する。山頂フィニッシュは区間順位と連動する
- ポイント賞の配点は公式値を維持し、平坦フィニッシュと中間スプリントの順位生成ではスプリント能力・加速力・スプリンター適性を強く評価する
- 平坦ゴールではゴール前位置を0〜100で判定し、アシストの高速リードアウト・最終発射とエースの位置取りカードを直接反映する。位置が悪い選手には包囲による勝負不参加リスクが発生する
- 勝負手「無賃乗車」は自前リードアウトと重複せず、最も強い他チームのリードアウト効果を50%借りて位置と発射出力へ反映する。強い他チーム列車がない場合は効果が小さい
- 個人総合用の安定スコアとステージ勝者用の勝負スコアを分離し、脚質適性、逃げ、展開分散、勝利後のマーク強化で一人の大量勝利を抑制する
- 個人総合トップ10は30分以内を目標帯とし、逃げ切り勝利は21ステージ中3〜4回。逃げ向きの丘陵・中山岳で非エースの脚質・闘争心・持久力・疲労から勝者候補を選ぶ
- 平坦ステージは着順と総合タイムを分離し、通常は集団同タイム。横風エシュロン、落車、機材故障などで切れた選手・チームだけにタイム差を付ける
- 超級山岳では登坂力82未満へグルペット時間を加算し、総合型／クライマー以外は総合を守らずステージ目標へ切り替える
- 上位アシスト2人の適性と残り体力でエースの疲労を軽減し、アシスト本人へ追加負荷を移す。疲労は総合・勝負スコアへ強く反映する
- ステージ別の獲得標高・勾配・風はゲーム用推定値として公式総獲得標高53,950mへ整合させ、公式値と推定値をデータ上で分離する
- グランツール各ステージには、総合・スプリント・山岳・クラシック・逃げ/ステージ狙い・TT狙いの方針別重要度を設定する
- 自動シミュレートでも省略ではなく、疲労・事故・順位・ポイント変動を処理する
- 課金設計として、シーズンパス、緩い選手スカウト、見た目課金、レース拡張パックを設定する
- スカウトは確率表示、天井、無料配布、重複時の汎用契約ポイント変換を前提にする。レベル上げ、限界突破、重複強化、期間限定人権は実装しない
- ゲーム独自組織「国際ロードサイクル連合（UCR）」と、年間ポイント／ライダー順位／チーム順位／対人レートの設定パネルを実装

## レース中コンディションの考え方

- `疲労`: 距離、高強度のアタック、登坂、横風、石畳・グラベルで蓄積する。持久力とチームの風よけで軽減する。
- `スプリント`: 加速、アタック、位置取りで消費する。隊列内での温存と十分な補給で一部回復する。
- `補給`: 糖質と水分をまとめたゲーム用指標。補給地点で回復し、低下するとハンガーノック域に入り、疲労と健康度を悪化させる。
- `健康度`: その日の体調、脱水耐性、悪路消耗をまとめた指標。疲労と補給不足が深刻になると出力を制限する。
- `バイクコントロール`: コーナリング、路面追従、下りのライン選択を表す。高いほど下り区間の出力が上がり、悪路の健康度低下と落車確率が下がる。

- `パヴェ`: 石畳を速く安全に走る専門能力。パリ〜ルーベの最高成績を主基準に、複数回の上位・完走実績を最大2点加味する。石畳ではパヴェ70%＋バイクコントロール30%で路面対応を判定し、グラベルには使用しない。
- `スプリント`: 5〜20秒程度の最高速。主にゴールスプリントの最大出力へ使う。
- `加速力`: ペース変化の鋭さ。アタック開始、位置取り、スプリント開始へ使う。
- `パンチ力`: 30秒〜5分程度の高出力。短坂、丘陵アタック、集団からの飛び出しに使う。
- `巡航力`: 数分以上にわたり高速を維持する力。平坦牽引、逃げ、個人TT、チームTTに使う。
- `持久力`: 長距離を走るためのエネルギー容量。距離による疲労増加を抑える。
- `耐性`: 高強度運動を繰り返す能力。アタックとスプリント時の疲労・乳酸負荷を抑える。
- `乳酸負荷`: アタック、短い登り、スプリントで増える高強度代謝負荷のゲーム表現。高いほど再加速とスプリント出力が落ちる。
- `回復力`: 流し、隊列内、補給、下りで乳酸負荷を下げ、スプリント残量を戻す速さ。丘陵の反復加速や登り後のフィニッシュで特に有効。
- `日間回復力`: 連続ステージ終了後の一晩の回復量。高いほど翌日へ持ち越す疲労・乳酸負荷が少なく、SP・健康度の回復量が多い。
- 乳酸は単なる疲労物質ではなくエネルギー代謝にも関わるため、実測血中乳酸値ではなく複合的な高強度負荷として抽象化する。
- 現実の研究をそのまま医療シミュレーションにせず、長時間走では糖質・水分摂取が重要であり、疲労により終盤出力が低下する関係をゲーム向けに抽象化している。

## アビリティ設計

- `パッシブ`: 選手固有の自動発動能力。編成、エース、山岳・横風などの条件に応じて、疲労、スプリント消費、補給効率、健康度、終盤出力を補正する。
- `アクティブ`: レース前に1つ選択し、対応する最初の勝負セクターで1回だけ発動する。発動時は疲労・スプリント・補給・健康度とセクター出力が同時に変化する。
- 実装済みアクティブは「全開アタック」「エース完全保護」「チーム補給」「最終発射」「横風分断」。コースと編成に合わない能力を選ぶと不発になる。
- 選手カードには固有パッシブ名を表示し、レース画面には発動中パッシブと選択中アクティブを表示する。

## 初期選手

選手カードは実在選手名を使用する。能力値は公開戦績と走行特性を基にしたゲーム用評価であり、現実の身体測定値や年俸を示すものではない。

初期300名は、固定の「勝利担当／アシスト系」では分けない。選手固有の `主脚質`・`副脚質` と、0〜100の `エース適性`・`アシスト適性` を持たせる。

役割は固定身分ではなく、現役300名の能力・実績・走り方から30種類の役割・戦術特性を最大7件まで付与する。エースとサブエースは原則として同時に付与しない。

8人編成は「エース候補2名＋支援役割6名」を基本形とするが、支援は選手の固定身分ではなく当日の任務とする。コース、体調、疲労、健康度、年間目標によって同じ選手の役割を変えられる。

初期300名は時代区分が「近代」「現代」の選手のみとし、「歴史」「クラシック」扱いの選手は後日登場するレジェンド別枠へ分離する。脚質としてのクラシック型は初期枠にも残す。

全300名に仮Credit年俸を設定する。8人チームの年俸上限はレース格に応じて通常42,000Cr、上位45,000Cr、最高峰48,000Crとし、主副脚質、両適性、固有能力、役割・戦術特性によって個別調整する。

世代最強級カードは例外枠として15,000Crまで設定する。Tadej Pogacarはエース適性99、アシスト適性84、15,000Crとし、Mathieu van der Poelはモニュメント8勝の最上位クラシック枠として13,000Crに設定する。

グランツール総合優勝が2回以上の選手は、勝利回数に応じてエース適性96〜98、Credit10,800〜13,000の最低保証を適用する。既存値は下げず、Tadej Pogacarのエース適性99・15,000Crは唯一の別格枠として維持する。
