# 選手スプレッドシート

ゲームで使用する選手データの正本です。人が探しやすいように、選手の状態別に3ファイルへ分けています。

| ファイル | 内容 | 現在の人数 |
|---|---|---:|
| `01_現役選手300名.csv` | 現役選手の所属、脚質、適性、役割、Credit、15能力 | 300名 |
| `02_引退選手.csv` | 引退選手の同項目 | 170名 |
| `03_区分保留.csv` | 現役・引退をまだ確定していない選手 | 40名 |

## 使い方

- 普段確認する場合は、最初に `01_現役選手300名.csv` を開きます。
- 選手の区分を変更するときは、該当するCSV間で行を移動します。
- 生成・集計ツールもこの3ファイルを直接参照します。別の場所にコピーしたファイルを正本として編集しないでください。
- `data/` には評価根拠、生成用JSON、旧ベースラインなどの機械処理・履歴データを残します。

## Googleスプレッドシート表示

- 役割は「役割（3つ）」の1列で表示します。
- 個人カードは結合せず、`basic_cards`（基本3枚）、`specialty_cards`（得意2枚）、`decisive_card`（勝負手1枚）の3列に分けます。
- カード割当は能力15項目、主副脚質、役割3つ、固有勝負手、高適性アシスト枠を照合し、`tools/assign_rider_cards.mjs` で全510選手へ再生成します。
- 割当の機械可読な正本は `data/rider_card_assignments.json` とし、手修正ではなく割当ルール側を直して再生成します。
- Googleスプレッドシート用の表示を更新するときも、右側のCredit・能力列を上書きせず、3カード列を独立して維持します。
