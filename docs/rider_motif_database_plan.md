# Rider Motif Database Plan

実在したロードレース選手を研究用モチーフとして整理し、ゲーム内では架空名・架空プロフィール・架空チームへ変換して使う。

## 基本方針

- 実名は開発用リファレンスに限定する。
- ゲーム本編には実名・肖像・所属チーム名・商標をそのまま出さない。
- 選手の価値は総合値だけでなく、脚質、性格、チーム相性、メンタルで分ける。
- 有名選手500人は、同じ強さの上位互換を並べるのではなく、役割が違うカードプールにする。
- 現役・近年選手は権利、肖像、炎上リスクが高いため、必ず架空化する。

## 初期300名の構成

初期300名は固定の「勝利担当／アシスト系」に分けない。選手固有の資質と、レース当日の任務を別々に管理する。

### 選手固有の軸

| 主脚質 | 人数 | 主な強み |
|---|---:|---|
| 総合型 | 11 | 長期総合、登坂、TT、日ごとの回復 |
| スプリンター | 30 | 平坦最高速、位置取り、発射台連携 |
| クライマー | 40 | 長い登坂、山岳耐久、軽量性 |
| パンチャー | 10 | 短い急坂、反復加速、丘陵決着 |
| クラシック型 | 50 | 石畳、横風、位置取り、悪天候 |
| TT・ルーラー型 | 59 | 巡航、単独走、集団牽引、隊列維持 |
| 合計 | 200 |  |

各選手は「主脚質」と「副脚質」に加え、0〜100の「エース適性」と「アシスト適性」を持つ。両適性は排他的ではなく、どちらも高い万能選手を認める。

### レースごとの役割

| 役割 | 内容 |
|---|---|
| 当日エース | チームの勝利・総合順位を最優先で狙う |
| 副エース | エースの不調や展開変化に備える第二勝負手 |
| アシスト | 牽引、補給、位置取り、風除け、落車復帰を担当 |
| ロードキャプテン | 隊列と戦術判断を統率する |
| リードアウト | スプリンターを最終加速へ運ぶ |
| ステージハンター | 特定ステージで勝利を狙う |
| 逃げ屋 | 逃げを成立・維持し、展開とポイントを動かす |

- 8人編成はエース候補2名＋支援役割6名を基本形とする。
- 支援は固定身分ではなく当日の任務であり、コース、体調、疲労、健康度、年間目標で変更できる。
- ステージハンター10名、逃げ屋10名を初期プールに確保する。同じ選手が他の役割を兼任できる。
- 別府史之モチーフはクラシック型／TT・ルーラー型でロードキャプテン適性を高くする。
- 新城幸也モチーフはクラシック型／パンチャーでアシスト、ロードキャプテン、逃げ支援に適性を持たせる。

### 初期300名の仮Credit年俸

- 仮Creditは「エースかアシストか」という固定身分では決めない。
- 主脚質、副脚質、両適性、固有パッシブ、回復力、健康度、年齢、専門役割を総合して価格を付ける。
- 8人チームの仮年俸上限は、通常レース42,000Cr、上位レース45,000Cr、最高峰レース48,000Crの3段階とする。
- 高額エース候補を複数採用すると残り枠が圧迫されるため、低〜中価格の高アシスト適性選手にも編成価値が生まれる。
- 選定した300名は専用ワークブックで管理し、実在選手名、国・地域、時代区分、主副脚質、両適性、得意役割、仮Credit、人物確認URLを記録する。
- 300名の設定上の正本一覧は docs/rider_motif_300_list.md とする。
- 歴史・クラシック選手の正本は docs/rider_motif_legacy_pool.md とし、初期300名に数えない。
## 500人へ拡張する場合の主脚質枠

500人へ拡張する場合も、役割を固定分類に戻さない。主脚質の目安だけを置き、当日役割は適性と編成で決める。

| 主脚質 | 目安人数 |
|---|---:|
| 総合型 | 70 |
| スプリンター | 80 |
| クライマー | 90 |
| パンチャー | 70 |
| クラシック型 | 90 |
| TT・ルーラー型 | 100 |
| 合計 | 500 |

ロードキャプテン、リードアウト、ステージハンター、逃げ屋などはこの人数と重複する役割タグとして設定する。

### 主脚質・副脚質・適性タグ

- 主脚質は最も得意な分野で、基礎能力、レースAIの選択、固有パッシブに強く影響する。
- 副脚質は次に得意な分野で、主脚質より弱い補正により対応コースを広げる。
- 第3脚質は通常設けず、パンチ力、ステージハンター、逃げ屋など最大3つの適性タグで局面適性を表す。
- 特別な万能選手は固有パッシブ「万能型」を持ち、第3分野に相当する適性タグのペナルティを副脚質に近い水準まで軽減する。
- 適性タグは主脚質の人数集計に含めず、レース役割とも分離して管理する。

## パラメータ化ルール

- グランツール総合勝者: `climb`, `stamina`, `recovery`, `team_dependency` を高める。
- ポイント賞・大量ステージ勝利系: `sprint`, `positioning`, `ego` を高める。
- 山岳賞系: `climb`, `fighting`, `attack_frequency` を高める。
- モニュメント/クラシック系: `technique`, `stamina`, `weather_resistance` を高める。
- TT系: `technique`, `stamina`, `mental_stability` を高め、`teamwork` は役割に応じて調整する。
- 高アシスト適性: `teamwork`, `protection`, `tactical_reading` を高める。本人の勝利性能は主副脚質とエース適性から独立して設定する。


## 支援役割を重視する編成ルール

- 8人チームは原則として当日エース1名、副エース候補1名、支援役割6名を割り当てる。
- エース気質の選手だけを並べると、横風、補給、位置取り、落車復帰、山岳列車、チームTTで大きく不利になる。
- 高エゴ選手は勝負所では強いが、複数入れすぎると隊列維持とエース保護が崩れる。
- 名アシストは年俸が低めでも勝利への貢献を大きくする。評価は希少度ではなく、年俸効率、役割適性、連携で決める。
- スカウトでは高年俸エースだけでなく、低〜中年俸の名アシスト、リードアウト、ロードキャプテンを当たり枠にする。
- グランツール総合狙いでは、山岳アシスト2人、平坦牽引1人、ロードキャプテン1人、TT/ルーラー1人を基本形にする。
- スプリントチームでは、リードアウト2人、横風キャプテン1人、平坦牽引2人が重要になる。

## Credit年俸方針

- 選手価値はランク文字では表記しない。
- 各選手には `Credit年俸` を設定し、ファンタジードラフトのように総年俸上限内で8人を組む。
- 最高峰エースは高年俸だが、複数並べるとアシスト不足とエゴ過多で勝率が落ちる。
- 名アシスト、リードアウト、ロードキャプテンは年俸効率が高く、グランツールや横風/石畳/TTTで価値が上がる。
- 年俸帯は目安として、世代最強級の例外枠 15,000、歴代級・最上位エース 12,000-14,999、トップエース 9,000-11,999、主力 6,000-8,999、中核・専門アシスト 3,000-5,999、低コスト枠 800-2,999 Credit。
- グランツール総合優勝が2回以上の選手には勝利回数別の最低保証を設ける。2-3勝はエース適性96・10,800Cr、4勝は98・12,000Cr、5-6勝は98・12,500Cr、7勝以上は98・13,000Crを下限とする。
- 補正は既存値を下げず、max（既存値, 最低保証）で適用する。ポガチャル型は例外としてエース適性99・15,000Crを維持する。
- 実績補正は加算せず、GT総合、個人ステージ、最終各賞、モニュメントのうち最も高い最低価格のみを採用する。Mathieu van der Poel型はモニュメント8勝の最低価格により13,000Crとする。
- 勝利数は2026年ジロ終了時点。参照: https://www.giroditalia.it/en/palmares/ / https://www.letour.fr/en/history / https://www.lavuelta.es/en/history
- 一覧表示ではツール、ジロ、ブエルタを別列にし、総合優勝1回につき★1個、未勝利は—で表す。3大会の★数合計はGT総合勝利数と必ず一致させる。
- 全300名について、ツール・ジロ・ブエルタ別に総合優勝、個人ステージ勝利、最終ポイント賞、最終山岳賞、最終新人賞を記録する。TTTは個人ステージ勝利に含めず、途中着用日数も賞の獲得回数に含めない。
- GT実績の基準日はツール2025年終了、ジロ2026年終了、ブエルタ2025年終了。進行中の大会は最終各賞が確定するまで加算しない。
- 全300名について五大モニュメント勝利数も大会別に記録し、1勝＝★1個、未勝利＝—で表示する。春の4大会は2026年終了、イル・ロンバルディアは2025年終了時点。
- 15,000 Creditカードは通常上限42,000 Creditの35.7%、最高峰上限48,000 Creditの31.3%を単独で占める。最高峰でも残り7枠は33,000 Creditに制限されるため、能力は別格でも編成難度が大きく上がる。
- 15,000 Creditと13,000 Creditを同時採用すると、最高峰上限48,000 Creditでも残り6枠は20,000 Creditとなり、二枚看板と支援層の配分が必要になる。
- 初期300名では3,000 Credit未満が0名のため、低価格帯は今後のカード追加または再評価で補う。
- 同じ総合値でも、主副脚質、適性タグ、エース適性、アシスト適性、専門役割、固有パッシブの組み合わせで年俸とコスト効率を変える。
- スカウト演出は「契約交渉」「年俸提示」「移籍市場」「新人契約」で表現する。

## 実装時の注意

- 実名をそのままカード名にしない。
- 顔、肖像、チームジャージ、スポンサー、商標は使わない。
- 現役選手モチーフは特に抽象化する。
- ドーピングや失格歴などセンシティブな実史は、ゲーム内では直接再現しない。
- スカウト時は「実在人物を引く」ではなく「架空選手と契約する」表現にする。
