# QOMISO Game Tour バトルカード・レースUI設計

更新日: 2026-07-20

## 1. 位置づけ

本書は次期バトル画面とカードシステムの採用設定をまとめた正本とする。現行HTMLプロトタイプは旧方式を含むため、実装更新時は本書を優先する。

## 2. 採用するバトル構造

- 選手固有カードをバトルの主役にする。
- 選手8人の選択が、そのまま個人カードデッキの構成になる。
- チームカードは個人カードと混ぜず、別デッキ・別UIで管理する。
- サポートカードと機材はレース前のパッシブ効果とし、レース中の第3の手札にはしない。
- 対CPU・対人とも、重要地点で作戦を選ぶ距離ベースのターン制とする。
- 対人は重要地点ごとに互いの作戦を伏せて選び、同時公開する。

## 3. カード設定

### 3.1 選手カード

選手カードは、固定能力、主副脚質、適性タグ、エース／アシスト適性、当日の役割、固有パッシブ、基本技、得意技、勝負手、Credit年俸を持つ。

レベル、経験値、覚醒、限界突破、重複による恒久強化は行わない。
主役割が総合エース、スプリントエース、丘陵エース、山岳エースの選手には、原則として二つ名と固有勝負手を1枚ずつ与える。固有勝負手は勝負手枠へ入り、コスト3、レース中1回とする。サブエースは自動付与の対象外とする。

固有勝負手を持たない選手は、主脚質・副脚質・役割に対応する共通勝負手を使用する。固有の有無だけでカードの基礎性能差は付けない。


### 3.2 個人カード

- 選手1人につき基本技3枚、得意技2枚、勝負手1枚の6枚を持つ。
- 8人編成では48枚の個人カードが使用候補になる。ランダムドローは行わず、選手を選ぶとその選手の6枚を表示する。
- 基本技3枚と得意技2枚は選手固有にせず、地形・役割に対応する共通カードから構成する。固有カードは原則として勝負手だけとする。
- 使用時はカードに描かれた選手本人の体力とSPを消費する。
- 1ターンに同じ選手が使える個人カードは1枚までとする。
- 作戦コスト内であれば、1ターンに複数選手の個人カードを使用できる。
- 同じ選手を繰り返し使うと、その選手が終盤に動けなくなる。
- 勝負手は原則レース中1回とする。

代表例:

- スプリンター: 前方位置キープ / 高速域からの再加速 / ファイナルスプリント
- クライマー: 一定勾配のリズム / 山岳ペースアップ / 頂上前アタック
- ルーラー: 平坦牽引 / エアロ巡航 / エアロ独走
- ロードキャプテン: 位置確認 / 逃げ差調整 / 隊列再編
- アシスト: エースの風よけ / 山岳牽引 / 最後の牽引
- リードアウト: 位置上げ / 高速リードアウト / 最終発射

### 3.3 アシストカード

- アシストカードも選手本人の6枚へ含め、個人カードとして扱う。
- アシスト適性80以上の選手には、有用なアシストカードを保証する。
- 基本技は選手固有にせず、4地形×2役割の24枚に役割別4枚を加えた共通基本カード全28枚とする。役割別基本技は先頭交代、ペース調整、高速コーナリング、安全誘導とする。
- 選手の基本技3枚は、当日の地形と役割に対応する3枚を固定で使う。
- 得意技も選手固有にせず、4地形ごとにエース用2枚、アシスト用4枚を用意する。共通得意技は全24枚とする。
- 選手の得意技2枚は地形別共通得意技と役割共通得意技から構成し、選手固有の得意技は原則として作らない。
- 通常の選手は、当日の役割に対応する22種類の役割共通カード（基本技4枚・得意技18枚）から基本技・得意技を構成する。
- 汎用勝負手は得意技の発展先として、エース用12枚・アシスト用12枚の全24枚を用意する。コスト3、レース中1回とし、固有勝負手との基礎威力差は付けない。
- アシスト用汎用勝負手は使用者の体力・SPを大きく消費し、本人ではなく味方エースまたはチームを強化する。
- 逃げ屋には逃げ参加、ブリッジ、先頭交代、ペース調整、逃げ選別、逃げ粘りの6枚を用意する。
- 下り用には高速コーナリング、下り加速、下りアタック、ダウンヒル牽引、安全誘導、下り復帰の6枚を用意する。
- 主エースと兼任する選手は、アシスト用の共通得意技を選択でき、固有勝負手は維持する。
- 主エースではない高適性アシストには、共通得意技に加えて固有の支援型勝負手1枚を保証する。
- 支援型勝負手はコスト3、レース中1回とし、自分の体力・SPを大きく消費して味方エースを強化する。
- アシストカードは、牽引、風よけ、位置上げ、追走、隊列再編、集団復帰、リードアウトを主な効果とする。
- 固有勝負手の有無だけで基礎性能差を付けず、アシスト選手を編成する戦術的価値を確保する。

### 3.4 作戦コスト

- 各ターンの作戦コストは原則5とする。
- 基本技は1、得意技は2、勝負手は3、チームカードは1〜2を消費する。
- 個人カードとチームカードの合計を、そのターンの作戦コスト以内に収める。
- 未使用の作戦コストは次ターンへ持ち越さない。
- 作戦コストはターンごとの指示量であり、選手本人が支払う体力・SPとは別資源とする。
- 選択したカードは行動キューへ並べ、プレイヤーが指定した順番で解決する。

### 3.5 チームカード

- レース前に6枚を選ぶ。
- ランダムドローではなく、選択可能な作戦を固定表示する。
- 使用回数、チームワーク消費、アシスト人数、クールダウンで連発を制限する。
- チームカードは1ターン最大1枚まで使用でき、個人カードと同じ作戦コストを消費する。
- 両カードの組み合わせでコンボを発生させる。

代表例: エース保護、集団追走、山岳列車、スプリント隊列、横風分断、逃げ容認、集団補給。

コンボ例: `山岳列車 + 頂上前アタック`、`スプリント隊列 + 爆発的フィニッシュ`。

### 3.6 カード選択と行動キュー

- ランダムな手札とカード交換は使用しない。
- 選手を選ぶと、その選手の基本技3枚、得意技2枚、勝負手1枚を表示する。
- 選んだカードは画面下部の行動キューへ追加する。
- 行動キューには実行順、選手、カード名、コスト、使用後の体力・SP予測を表示する。
- 決定前ならカードの取り消しと並べ替えができる。
- 作戦コストを超えた組み合わせは決定できない。

## 4. ターン進行

1. 重要地点、標高、風向き、集団差を確認する。
2. 必要ならチームカーからチームカードを1枚選び、行動キューへ追加する。
3. 選手を選び、作戦コスト内で複数選手の個人カードを1人1枚まで行動キューへ追加する。
4. 実行順と、使用後の体力・SP予測を確認する。
5. 合計作戦コストが5以内であることを確認して作戦を決定する。
6. CPUまたは対戦相手の作戦と同時公開し、行動キュー順に解決する。
7. コース上で結果を再生し、集団位置、タイム差、選手状態、実況を更新する。

200km級でも直接選択するのは6〜8か所の重要地点を基本とする。通常巡航区間は自動処理する。

代表地点: スタート直後、横風・悪路入口、逃げへの対応、補給、山岳入口、残り10km、フィニッシュ。

## 5. メインバトル画面

### 5.1 地図

- 淡色で簡略化した、斜め上からの俯瞰地図を主画面にする。
- コースは太い一本のラインで示す。
- 常時表示は `START`、`GOAL`、`逃げ集団`、`メイングループ` を基本とする。
- 独立した「現在地」マーカーは表示しない。走行中の集団そのものを現在位置とする。
- 選択中選手は地図上でも輪郭や光で対応表示できる。
- 集団間のタイム差は実況または集団ラベルの補助情報として示す。

### 5.2 標高と風

- 標高プロファイルは画面上部中央に細く表示する。
- 平坦、丘陵、登り、下りと、おおよその区間距離を読めるようにする。
- 風向きと風速は右上へ固定表示する。
- 風は矢印と `横風 24km/h` のような短い表記を使う。
- 標高と風はカード効果の判断材料として使う。

### 5.3 チームカーとメンバー

- 左端にチームカーと選手8人を縦に並べる。
- 最上段をチームカー、その下を選手8人とする。
- チームカーを選ぶとチームカードを開く。
- 選手を選ぶと、その選手が使用可能な個人カードを開く。
- 選択中の対象はシアンの枠で強調する。
- 選手へ切り替えても、選択済みチームカードを失わない。

### 5.4 選手状態

全選手一覧ではバーを2本だけ常時表示する。

- `体力`: 長距離を走り続ける余力。多いほど良い。
- `SP`: アタック、位置取り、再加速、スプリントに使う短期資源。多いほど良い。

疲労は内部で増加量として保持するが、一覧では反転した体力として表示する。2本とも「多いほど良い」に統一する。

常時バーを増やす代わりに、補給不足、水分不足、高強度負荷、落車・不調、集団からの遅れ、牽引中、エース保護中を状態アイコンで示す。

選手を選んだときだけ、補給、健康度、高強度負荷、回復力、現在の役割などの詳細を表示する。

### 5.5 カード配置と行動キュー

- 選択中選手の6枚を画面下部中央へ、3枚ずつの2段または横一列で並べる。
- 大きいアイコン、短いカード名、効果または消費の予測値を表示する。
- 選択中カードだけを上へ持ち上げ、拡大し、黄色枠で強調する。
- マウスでもタッチでも選びやすい大きさにする。
- 地図へ少し重ねてよいが、コースと集団を隠さない。
- チームカードと個人カードは色、見出し、選択元で区別する。
- 行動キューはカード列の上に表示し、合計作戦コストを `3 / 5` のように常時表示する。
- 行動キューのカードはドラッグまたはボタンで実行順を変更できる。

パワプロの戦術試合に見られる「下部横並び・選択中を強調する」情報構造を参考にする。固有の絵、意匠、配色、数値体系は複製しない。

### 5.6 実況

- 実況は画面の絶対最下段へ、全幅の1行テキストバーとして表示する。
- カード列より下に置く。
- 集団差、風、アタック、追走、トラブル、次の勝負所を短文で伝える。
- 重要な変化から優先し、ログの羅列にはしない。

例: `残り42km。横風区間に入り、メイングループが追走を開始します。`

## 6. 内部パラメーター

一覧のバーは2本だが、内部計算では疲労、スプリント残量、補給、健康度、高強度負荷、回復力、集団内位置、当日の役割、チームワーク補正を保持する。

乳酸は実測値ではなく、アタック、短い登り、スプリント反復による複合的な高強度負荷として抽象化する。

カードへカーソルを合わせるかタップしたときは、使用後の体力とSPをバー上にプレビューする。

## 7. チームウェア規則

### 7.1 通常時

- 同一チームは登録された同一デザインのチームジャージを着る。
- ヘルメットも原則としてチームカラーと基本デザインを統一する。
- 選択や役割のためにジャージ・ヘルメット色を変えない。
- 選手は顔、髪、表情、名前、番号、役割アイコン、選択枠で識別する。

### 7.2 例外ジャージ

現実のロードレース規則に合わせ、現在資格を持つ選手だけ通常チームジャージから変更する。

- ステージレースの分類リーダー
- 現世界王者
- UCIカップ・シリーズ・ランキングの現リーダー
- 現大陸王者
- 現ナショナル王者
- ナショナルチーム参加時の代表ジャージ

過去に賞を獲得しただけでは変更しない。

複数資格の優先順位:

1. ステージレースのリーダージャージ
2. 世界王者ジャージ
3. UCIカップ・シリーズ・ランキングのリーダージャージ
4. 大陸王者ジャージ
5. ナショナル王者ジャージ
6. ナショナルチームジャージ

ステージレース内の複数首位は、個人総合時間、ポイント、山岳、ヤングなどその他分類の順とする。主催者設定により、着られない下位分類ジャージを次順位選手へ着用させられる。チーム総合首位には専用ジャージを与えず、必要なら特別ゼッケンで表現する。

ナショナル王者ジャージは獲得した種目・専門・カテゴリーで、次回大会開始前日まで着用する。ロード王者と個人TT王者は別資格とする。元王者は該当種目で、襟や袖へ国旗色のパイピングを表示できる。

ヘルメットの賞・王者カラーは任意の視認演出とし、ジャージ資格判定と分ける。

## 8. UI参考方針

- Pro Cycling Manager系から、標高、風、集団、全メンバーのエネルギーを同時に読む情報階層を参考にする。
- パワプロ栄冠ナイン系から、下部に戦術カードを横並びにし、選択中を大きく強調する操作性を参考にする。
- 参考作品の画面、ロゴ、カード絵、配色、固有アイコン、数値体系は複製しない。
- 本作では地図を主役にし、判断に必要な情報だけ常時表示する。

## 9. 実装時の最低受け入れ条件

- START、GOAL、逃げ集団、メイングループを地図に表示する。
- 独立した現在地マーカーを置かない。
- 標高プロファイルと風向き・風速を表示する。
- チームカーと選手8人を左端に縦表示する。
- 全選手に体力・SPの2本のバーを表示する。
- 同一チームの通常選手は同一ジャージ・同一系統ヘルメットを着用する。
- チームカー選択時はチームカード、選手選択時は個人カードを開く。
- 選択中選手のカード6枚を下部に表示し、選択カードを持ち上げて強調する。
- 作戦コスト5、1選手1枚、チームカード1枚までの制約を実装する。
- 複数選手のカードを行動キューへ追加し、実行順と合計コストを表示する。
- 実況を絶対最下段に表示する。
- チームカードと個人カードを別状態で保持し、組み合わせて決定できる。
- 個人カード使用後、実行選手本人の体力・SPが変化する。
